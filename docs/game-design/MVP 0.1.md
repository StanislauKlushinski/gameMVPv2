# MVP 0.1

## Цель MVP

Первый MVP должен проверить, работает ли базовая fantasy игры без перегруза мета-системами:
- у игрока есть 2 эпохи, работающие параллельно
- в каждой эпохе есть короткая производственная цепочка
- игрок видит bottleneck и устраняет его строительством и апгрейдами
- одна эпоха помогает другой через простой межэпохный канал
- у сессии есть короткая достижимая цель

## Что входит

- 2 параллельные эпохи
- по 1 короткой цепочке производства в каждой эпохе
- по 3-4 ресурса в каждой эпохе
- по 3-4 здания в каждой эпохе + **1 здание CycleGenerator в каждой эпохе**
- общий пул ресурсов внутри каждой эпохи
- автоматическое распределение ресурсов между зданиями
- приоритеты зданий и понятный простой при дефиците
- апгрейды зданий
- минимальное дерево технологий на 5-7 узлов
- 1 межэпохный ресурсный канал
- 1 числовой кросс-эпохный бонус
- **рабочий цикл каждой эпохи → глобальный счётчик `globalWorkCycles`**
- **панель GlobalCyclePanel с прогресс-баром к порогу престижа**
- короткая финальная цель первой версии (требует минимум globalWorkCycles)

## Что не входит

- полноценный престиж
- полный оффлайн-прогресс
- отдельная система глобальных улучшений
- полная ракета как финал цикла
- сложные графы межэпохных связей
- глубокая система рабочих и ручное распределение рабочих
- сложная логистика с несколькими маршрутами, буферами и развилками

## Эпоха 1. Каменный век

**Роль в MVP:**
- стартовая эпоха
- даёт базовые ресурсы и первый экспорт
- учит игрока читать bottleneck на простом цикле

**Ресурсы:**
- дерево
- камень
- доски
- инструменты
- **Рабочий цикл каменного века** (tier = EpochOutput; не в пуле эпохи)

**Здания:**
- Лесоруб
- Каменоломня
- Мастерская
- Перевалочный склад
- **Кузня циклов** (CycleGenerator: потребляет инструменты → производит рабочие циклы)

**Цепочка:**
- дерево → доски
- доски + камень → инструменты
- инструменты → **рабочий цикл** (через Кузню циклов)

## Эпоха 2. Индустриальная эпоха

**Роль в MVP:**
- вторая производственная зона
- использует более дорогую переработку
- зависит от экспорта из первой эпохи и затем усиливает её

**Ресурсы:**
- руда
- уголь
- металл
- машинные детали
- **Промышленный рабочий цикл** (tier = EpochOutput; не в пуле эпохи)

**Здания:**
- Шахта
- Плавильня
- Сборочный цех
- Логистический узел
- **Промышленный агрегатор** (CycleGenerator: потребляет машинные детали → производит рабочие циклы)

**Цепочка:**
- руда + уголь → металл
- металл + инструменты → машинные детали
- машинные детали → **промышленный рабочий цикл** (через Промышленный агрегатор)

## Межэпохная связь

В первом MVP оставить только 2 типа связи:

1. **Ресурсный канал**
- `инструменты` из Каменного века могут передаваться в Индустриальную эпоху
- передача идёт через общий лимит межэпохной логистики
- если лимит забит, игрок получает понятный bottleneck

2. **Числовой бонус**
- после открытия индустриальной технологии `Механизация добычи` Каменный век получает `+15%` к добыче базовых ресурсов
- бонус должен быть виден в интерфейсе как эффект второй эпохи на первую

## Минимальное дерево технологий

Для первой версии достаточно 6 технологий:
- Обработка дерева
- Каменные инструменты
- Межэпохная доставка
- Добыча руды
- Плавка металла
- Механизация добычи

## Игровая цель первой версии

Вместо полной ракеты MVP заканчивается на короткой многошаговой цели:
- произвести стабильный поток инструментов в Каменном веке
- запустить экспорт инструментов в Индустриальную эпоху
- наладить выпуск машинных деталей
- построить `Прототип пускового модуля` (требует минимального количества `globalWorkCycles`)

`Прототип пускового модуля` — это короткая вертикальная цель, которая:
- требует ресурсы из двух эпох
- требует, чтобы оба CycleGenerator работали и накопили минимум глобальных циклов
- проверяет межэпохную связь
- при выполнении — разблокирует кнопку престижа и открывает экран `PrestigeScreenController`

## Числа в MVP

Все игровые количества реализуются через тип `GameNumber`:
- `ResourceAmount.amount: GameNumber`
- `ResourcePoolState.amounts: Dictionary<string, GameNumber>`
- `GameState.globalWorkCycles: GameNumber`

MVP-упрощение: внутри `GameNumber` допустимо использовать `double`. Когда числа приблизятся к `1e308` — меняется только внутренность типа, весь остальной код без изменений.

## Критерии готовности MVP

MVP считается успешным, если:
- ресурсы генерируются автоматически
- здания потребляют и производят ресурсы
- есть лимиты хранения и лимит межэпохной передачи
- есть хотя бы 1 bottleneck внутри эпохи и 1 bottleneck между эпохами
- игрок может строить и улучшать здания
- игрок понимает, почему производство остановилось
- у каждой эпохи есть свои ресурсы, здания и независимая цепочка
- межэпохный экспорт инструментов реально влияет на вторую эпоху
- бонус `Механизация добычи` реально ускоряет первую эпоху
- **в каждой эпохе работает CycleGenerator и производит рабочие циклы**
- **`globalWorkCycles` растёт и отображается в GlobalCyclePanel**
- **при выполнении финальной цели (включая минимум globalWorkCycles) открывается экран престижа**
- все числовые значения ресурсов используют `GameNumber`, не `float`

## Зачем такой MVP

Этот MVP должен ответить на один главный вопрос:

интересно ли игроку управлять двумя связанными производственными слоями, искать bottleneck и усиливать одну эпоху через другую, даже если все мета-системы пока отключены.
