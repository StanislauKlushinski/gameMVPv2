# Связь эпох (кросс-эффекты)

Связь эпох (кросс-эффекты)

### 15.1. Определение

Связь эпох — система, которая позволяет ресурсам, эффектам и бонусам из одной эпохи влиять на другие эпохи.

Это слой "межсистемной интеграции", который соединяет все производственные эпохи в единую экономику.

### 15.2. Типы связей

#### 15.2.1. Ресурсные связи

Epoch A -> Epoch B (resource transfer)

**Пример:**

дерево из каменного века используется в индустриальной эпохе

#### 15.2.2. Производственные бонусы

Epoch A production boosts Epoch B

**Пример:**

индустриальная автоматизация ускоряет добычу дерева

#### 15.2.3. Технологические связи

tech unlock in A affects B

#### 15.2.4. Глобальные эффекты

epoch contributes to global multiplier

### 15.3. Модель связи

```text
CrossEpochLink {from_epoch: Epochto_epoch: Epochtype: enumstrength: floatdecay: float}
```

### 15.4. Сила связи

```text
effective_strength = base_strength * modifiers
```

### 15.5. Ослабление связи

**Чтобы избежать доминирования одной эпохи:**

```text
final_effect = strength * 0.2 .. 0.7
```

### 15.6. Направленные связи

**Связи не обязательно двусторонние:**
- A -> B разрешено
- B -> A может отсутствовать

### 15.7. Обмен ресурсами

#### 15.7.1. Прямой

```text
transfer(resource, epoch_A, epoch_B)
```

#### 15.7.2. Через буфер

global_buffer -> distribution system

### 15.8. Производственные усиления

**Пример формулы:**

production_B *= (1 + output_A * coefficient)

### 15.9. Кросс-цепочки

**Позволяют строить:**
- цепочки между эпохами
- многоуровневые рецепты

### 15.10. Ограничения

#### 15.10.1. Лимит влияния

max_cross_epoch_bonus

#### 15.10.2. Убывающая отдача

```text
effectiveness = log(x) or sqrt(x)
```

### 15.11. Балансная модель

**Связи должны:**
- усиливать стратегию
- не заменять локальные цепочки
- создавать выбор

### 15.12. Риски системы

Одна эпоха становится "главной"

Слишком сложная оптимизация

Потеря читаемости экономики

Непредсказуемые петли ресурсов

### 15.13. UI требования

**Игрок должен видеть:**
- граф связей между эпохами
- входящие и исходящие эффекты
- силу влияния
- bottleneck между эпохами

### 15.14. Минимальный набор (definition of done)

**Система считается реализованной, если:**
- есть хотя бы 2 типа кросс-связей
- ресурсы могут пересекать эпохи
- есть влияние одной эпохи на другую
- есть ограничения силы эффектов
- игрок может это понимать через UI
