# Здания (типы и функции)

Здания (типы и функции)

### 4.1. Определение

**Здание — основная производственная единица, которая:**
- генерирует ресурсы
- преобразует ресурсы
- влияет на систему (логистика, лимиты, прогрессия)

### 4.2. Базовая структура

```text
Building {id: stringname: stringtype: enumepoch: referencelevel: intmax_level: intbase_production: floatrecipe: Recipe | nullworker_slots: intinternal_storage: map<resource, float>priority: intis_active: bool}
```

### 4.3. Типы зданий

#### 4.3.1. Добывающие

**Функция:**

генерация базовых ресурсов

**Особенности:**
- не требуют входных ресурсов
- работают постоянно

**Пример поведения:**

output += base_rate * multipliers

#### 4.3.2. Производственные

**Функция:**

переработка ресурсов

**Особенности:**
- имеют рецепт
- требуют входные ресурсы
- if input >= required: consume input
- produce output

#### 4.3.3. Жилые

**Функция:**
- увеличивают лимит рабочих
- max_workers += value

#### 4.3.4. Логистические

**Функция:**

усиливают потоки ресурсов

**Эффекты:**
- скорость передачи
- лимиты
- буферы

#### 4.3.5. Технологические

**Функция:**

открытие контента

**Эффекты:**
- новые здания
- новые ресурсы
- новые эпохи

### 4.4. Поведение здания

#### 4.4.1. Состояния

ACTIVEIDLE (нет ресурсов)BLOCKED (лимит)DISABLED

#### 4.4.2. Условия работы

```text
is_active = (есть входные ресурсы)AND (есть место в складе)AND (включено игроком)
```

### 4.5. Уровни зданий

#### 4.5.1. Рост параметров

```text
production = base * (growth_factor ^ level)
```

#### 4.5.2. Стоимость улучшения

```text
upgrade_cost = base_cost * (cost_factor ^ level)
```

### 4.6. Ограничения

#### 4.6.1. Лимит количества

```text
max_instances_per_type = 3 (на эпоху)
```

#### 4.6.2. Лимит уровня

```text
max_level = N
```

### 4.7. Взаимодействие с ресурсами

#### 4.7.1. Вход

pull from resource pool

#### 4.7.2. Выход

push to resource pool

#### 4.7.3. Внутренний буфер

internal_storage <= internal_limit

### 4.8. Приоритет

**Каждое здание:**

priority: int

**Влияет на:**
- получение ресурсов
- порядок обработки

### 4.9. Включение/выключение

**Игрок может:**

```text
toggle(building)
```

**Назначение:**
- управление потоками
- устранение bottleneck

### 4.10. Назначение рабочих

assigned_workers <= worker_slots

**Эффект:**

production *= worker_multiplier

### 4.11. UI требования

**Игрок должен видеть:**
- уровень здания
- текущий статус
- вход/выход ресурсов
- эффективность (%)
- количество рабочих
- апгрейд стоимость

### 4.12. Балансные параметры

**Каждое здание имеет:**

base_rategrowth_factorcost_factorworker_effect

### 4.14. Минимальный набор (definition of done)

**Система зданий реализована, если:**
- есть минимум 2 типа зданий
- есть производство и переработка
- есть уровни
- есть апгрейды
- есть взаимодействие с ресурсами
- здания могут простаивать
