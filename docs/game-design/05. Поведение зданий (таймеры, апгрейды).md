# Поведение зданий (таймеры, апгрейды)

Поведение зданий (таймеры, апгрейды)

### 5.1. Определение

**Поведение здания — это логика, по которой оно:**
- потребляет ресурсы
- обрабатывает время
- производит результат
- реагирует на ограничения

### 5.2. Модель работы (основа)

**Каждое здание работает по циклу:**

## 1. Проверка условий2. Потребление ресурсов3. Обработка времени4. Производство результата5. Переход к следующему циклу

### 5.3. Типы производственного поведения

#### 5.3.1. Непрерывное (continuous)

**Ресурсы производятся плавно:**

output += rate * delta_time

**Используется для:**
- добычи
- простых производств

#### 5.3.2. Дискретное (tick/cycle)

**Производство по завершению цикла:**

если timer >= cycle_time:произвести outputtimer = 0

**Используется для:**
- сложных рецептов
- крафта

### 5.4. Таймер здания

timer += delta_time * speed_multiplier

### 5.5. Скорость работы

```text
speed_multiplier = worker_bonus *building_level_bonus *global_bonus
```

### 5.6. Потребление ресурсов

#### 5.6.1. Для дискретной модели

при старте цикла:списать input ресурсы

#### 5.6.2. Для непрерывной модели

input -= rate * delta_time

### 5.7. Проверка запуска

**Здание может начать цикл если:**

input >= requiredAND output_storage_not_full

### 5.8. Буфер входа

**Каждое здание:**

internal_storage[resource]

**Используется для:**
- хранения входных ресурсов
- предотвращения "дёргания"

### 5.9. Буфер выхода

output_buffer

**Если:**

output_buffer >= limit:stop production

### 5.10. Состояния здания

```text
ACTIVEWAITING_FOR_INPUTWAITING_FOR_OUTPUT_SPACEDISABLED
```

### 5.11. Прерывание цикла

**Если ресурсы закончились:**
- Вариант (рекомендуемый):
- цикл не начинается

**Но если уже начался:**
- цикл завершается
- (ресурсы уже списаны)

### 5.12. Параллельные циклы

**Если здание может:**

```text
parallel_jobs = N
```

**Тогда:**

timer[i] для каждого слота

### 5.14. Эффект рабочих

```text
worker_bonus = 1 + workers * factor
```

**Ограничение:**

max_workers_per_building

#### 5.13.2. Производство

output *= (1 + level_multiplier)

### 5.13. Эффект уровня

#### 5.13.1. Скорость

cycle_time /= (1 + level_bonus)

### 5.15. Апгрейды здания

**Типы:**
- Скорость
- Объём производства
- Эффективность (меньше затрат)
- Буфер

### 5.16. Формула апгрейда

```text
new_value = base_value * (upgrade_multiplier ^ level)
```

### 5.17. Включение/выключение

```text
is_active = true/false
```

**Если выключено:**
- таймер не идёт
- ресурсы не потребляются

### 5.18. Поведение при переполнении

если нет места:pause

### 5.19. Поведение при оффлайн прогрессе

**Упрощённая модель:**

```text
cycles = floor(offline_time / cycle_time)
```

```text
total_output = cycles * output
```

### 5.20. Приоритет обновления

**Порядок:**

## 1. Проверка условий2. Запуск новых циклов3. Обновление таймеров4. Завершение циклов

### 5.21. Требования к реализации

Без накопления ошибок таймера

Поддержка больших скоростей

Стабильность при delta_time скачках

Детерминированность

### 5.22. Ошибки проектирования

Частичное потребление (сложная логика)

Зависимость от FPS

"Застревание" таймера

Непрозрачное состояние

### 5.23. Минимальный набор (definition of done)

**Система считается реализованной, если:**
- есть цикл производства
- есть таймер
- есть вход/выход ресурсов
- здание может останавливаться
- есть влияние апгрейдов
- есть влияние рабочих
