# Престиж (мета-прогресс)

Престиж (мета-прогресс)

### 10.1. Определение

**Престиж — мета-механика, при которой игрок:**
- сбрасывает текущий прогресс
- получает постоянные бонусы
- ускоряет следующие циклы

### 10.2. Условия престижа

**Престиж доступен при выполнении одного из условий:**

## 1. достигнут порог ресурса2. открыта финальная технология3. построен ключевой объект (ракета)

### 10.3. Процесс престижа

## 1. игрок активирует престиж2. вычисляется награда3. происходит сброс4. применяются бонусы5. начинается новый цикл

### 10.4. Что сбрасывается

- ресурсы- здания- уровни зданий- рабочие- локальные технологии

### 10.5. Что сохраняется

- престижная валюта- глобальные улучшения- открытые мета-системы

### 10.6. Награда за престиж

#### 10.6.1. Валюта престижа

prestige_points += earned_points

#### 10.6.2. Формула начисления

```text
prestige_points = f(total_production)
```

**Рекомендуемый вариант:**

```text
prestige_points = sqrt(total_resources_produced)
```

**или:**

```text
prestige_points = log(total_resources)
```

### 10.7. Глобальные бонусы

**Престиж даёт:**

#### 10.7.2. Ускорение

production_speed += value

#### 10.7.3. Логистика

transfer_rate += bonus

#### 10.7.4. Кросс-эпохные эффекты

cross_epoch_modifier += value

#### 10.7.1. Множители

global_multiplier *= (1 + bonus)

### 10.8. Престижное дерево

**Игрок тратит валюту:**

```text
PrestigeUpgrade {costeffectlevel}
```

**Типы:**
- скорость
- эффективность
- лимиты
- новые механики

### 10.9. Масштабирование

**Стоимость апгрейдов:**

```text
cost = base * (scale ^ level)
```

### 10.10. Оптимальный момент престижа

**Игрок должен решать:**

wait for more points vs reset now

### 10.11. diminishing returns

gain_efficiency decreases over time

**Назначение:**

мотивировать делать престиж

### 10.12. Частота престижа

**Целевые интервалы:**
- ранняя игра: часто
- средняя: умеренно
- поздняя: редко

### 10.13. UI требования

**Игрок должен видеть:**
- сколько получит валюты
- что потеряет
- текущие бонусы
- прогноз после престижа

### 10.14. Баланс

**Престиж должен:**
- Сильно ускорять старт
- Давать ощутимый прогресс
- Не ломать баланс полностью

### 10.15. Ошибки проектирования

Слишком слабый престиж

Слишком сильный (ломает игру)

Непонятная формула

Отсутствие выбора

### 10.16. Минимальный набор (definition of done)

**Система реализована, если:**
- есть возможность сброса
- есть расчёт награды
- есть глобальные бонусы
- есть повторяемый цикл
- есть ощущение ускорения
