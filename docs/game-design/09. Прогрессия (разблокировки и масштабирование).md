# Прогрессия (разблокировки и масштабирование)

Прогрессия (разблокировки и масштабирование)

### 9.1. Определение

**Прогрессия — система, определяющая:**
- темп развития игрока
- порядок открытия контента
- рост сложности и масштабов

### 9.2. Уровни прогрессии

#### 9.2.1. Локальная (внутри эпохи)

**Игрок:**
- открывает здания
- строит цепочки
- увеличивает производство

#### 9.2.2. Межэпохная

**Игрок:**
- открывает новые эпохи
- добавляет новые слои производства

#### 9.2.3. Мета-прогрессия

**Игрок:**
- делает престиж
- ускоряет будущие циклы

### 9.3. Основная модель роста

**Прогрессия основана на:**

production -> reinvestment -> higher production

### 9.4. Кривые роста

#### 9.4.1. Производство

```text
production = base * (scale ^ time)
```

#### 9.4.2. Стоимость

```text
cost = base * (scale ^ level)
```

#### 9.4.3. Баланс

**Условие:**
- growth_production > growth_cost
- иначе игра "умирает".

### 9.5. Разблокировки

**Типы:**
- Здания
- Ресурсы
- Эпохи
- Улучшения

### 9.6. Пороговые условия

**Разблокировка происходит при:**
- resource >= threshold
- или
- technology_unlocked

### 9.7. Темп игры

#### 9.7.1. Фазы

Быстрый старт

Замедление

Ускорение через апгрейды

Плато

Престиж

#### 9.7.2. Цель

**Игрок должен:**
- регулярно получать новые возможности
- не застревать надолго

### 9.8. Узкие места как двигатель

**Прогресс строится через:**

solve(bottleneck) -> create_new_bottleneck

### 9.9. Введение новых механик

**Новые системы вводятся:**
- через технологии
- через эпохи

**Правило:**

не вводить несколько сложных систем одновременно

### 9.10. Масштабирование сложности

#### 9.10.1. По эпохам

больше ресурсов

сложнее рецепты

длиннее цепочки

#### 9.10.2. По времени

растущие стоимости

растущие требования

### 9.11. Кросс-эпохная прогрессия

older_epoch_bonus += new_epoch_effect * modifier

### 9.12. Контроль перегрузки

**Ограничения:**
- лимит зданий
- лимит рабочих
- лимит логистики

### 9.13. Контроль скорости

**Инструменты:**
- стоимость
- лимиты
- требования технологий

### 9.14. Мягкие блокировки

slow_progress instead of hard_lock

**Игрок:**
- может двигаться дальше
- но медленно

### 9.15. Метрики баланса

**Нужно контролировать:**
- время до следующего апгрейда
- рост производства
- частоту решений игрока

### 9.16. Ошибки проектирования

Слишком быстрый рост

Полная остановка прогресса

Слишком резкие скачки

Отсутствие целей

### 9.17. Минимальный набор (definition of done)

**Система реализована, если:**
- есть рост производства
- есть рост стоимости
- есть разблокировки
- есть хотя бы 1 эпоха
- есть хотя бы 1 bottleneck
