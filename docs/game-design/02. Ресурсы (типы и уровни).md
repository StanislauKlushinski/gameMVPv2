# Ресурсы (типы и уровни)

Ресурсы (типы и уровни)

### 2.1. Определение

**Ресурс — числовая сущность, используемая для:**
- производства
- строительства
- прогрессии

### 2.2. Классификация ресурсов

#### 2.2.1. По уровню

## 1. Базовые

добываются напрямую

не требуют входных ресурсов

**Примеры:**
- дерево
- камень
- руда

## 2. Промежуточные

создаются из базовых или других промежуточных

**Примеры:**
- доски
- кирпич
- инструменты

## 3. Продвинутые

используются в поздних цепочках

**Примеры:**
- механизмы
- электроника
- топливо

### 2.3. Формальная модель ресурса

**Каждый ресурс описывается структурой:**

```text
Resource {id: stringname: stringtype: enum (base, intermediate, advanced)epoch: referencecurrent_amount: floatstorage_limit: float}
```

### 2.4. Хранение ресурсов

#### 2.4.2. Ограничение хранения

amount <= storage_limit

**Если превышение:**

```text
production = 0
```

#### 2.4.1. Локальное хранилище эпохи

**Каждая эпоха имеет свой пул:**

epoch_resources[resource_id] = amount

### 2.5. Генерация ресурсов

**Происходит через здания:**

```text
delta_resource = production_rate * delta_time
```

**Добавление:**
- resource += delta_resource
- С учётом лимита.

### 2.6. Потребление ресурсов

if resource >= required:resource -= requiredelse:stop_production

### 2.7. Рецепты

Ресурсы преобразуются через рецепты.

**Структура:**

```text
Recipe {input: {resource_id: amount}output: {resource_id: amount}time: float}
```

### 2.8. Скорость производства ресурса

```text
output_per_sec = output_amount / recipe_time * total_multiplier
```

### 2.9. Баланс ресурсов

#### 2.9.2. Избыточный ресурс

**Если:**

production >> consumption

**то ресурс:**
- перестаёт быть значимым
- должен использоваться дальше

#### 2.9.1. Узкое место

```text
bottleneck = min(production_rate(resource_i))
```

### 2.10. Связь ресурсов между эпохами

#### 2.10.2. Ослабление эффекта

```text
effective_value = base_value * cross_epoch_modifier
```

**где:**

```text
cross_epoch_modifier = 0.1 - 0.3
```

#### 2.10.1. Правило доступа

**Ресурс может:**
- использоваться внутри своей эпохи
- использоваться в других эпохах (если разрешено)

### 2.11. Масштабирование чисел

**Idle требует роста:**

```text
value = base * (growth_factor ^ level)
```

**Требования:**

поддержка больших чисел (double или BigNumber)

**отображение:**

K, M, B, T и т.д.

### 2.12. Приоритеты потребления

**Если ресурс используется в нескольких местах:**
- Варианты:
- равномерно
- по приоритету
- по порядку

**Рекомендуется:**

```text
priority_system = true
```

(игрок может задавать приоритет зданий

### 2.13. Дефицит ресурса

**Если ресурс заканчивается:**
- здание полностью останавливается
- частичная работа не допускается (упрощение логики)

### 2.14. UI требования

**Игрок должен видеть:**
- текущее количество
- скорость (+X/сек)
- лимит хранения
- потребление (-Y/сек)
- дефицит (индикатор)

### 2.15. Ошибки, которых нужно избегать

Ресурс без использования

Бесконечный избыток без лимита

Непрозрачное потребление

Слишком сложные рецепты на старте

### 2.16. Минимальный набор (definition of done)

**Система ресурсов считается реализованной, если:**
- есть минимум 3 ресурса
- есть хотя бы 1 цепочка (A -> B -> C)
- есть лимит хранения
- есть производство и потребление
- есть отображение скорости
