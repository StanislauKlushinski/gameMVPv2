# Эпохи (параллельные слои)

Эпохи (параллельные слои)

### 14.1. Определение

Эпоха — независимый производственный слой со своей экономикой, зданиями и ресурсами, работающий параллельно с другими эпохами.

### 14.2. Ключевая идея

эпохи существуют одновременно

игрок не "переходит", а "добавляет слой"

каждая эпоха остаётся активной до конца игры

### 14.3. Структура эпохи

```text
Epoch {
  id: string
  name: string
  resources: list<Resource>          // включая один ресурс с tier = epoch_output
  buildings: list<Building>          // включая одно здание типа CycleGenerator
  technologies: list<Technology>
  is_unlocked: bool
  epoch_cycle_resource_id: string    // ID ресурса-цикла этой эпохи (tier = epoch_output)
}
```

**epochCycleResourceId** — ключевое поле эпохи. Оно связывает эпоху с её итоговым производственным выходом. Здание типа `CycleGenerator` потребляет продвинутые ресурсы цепочки и производит этот ресурс, который идёт в глобальный счётчик `globalWorkCycles`.

### 14.4. Примеры эпох

Каменный век

Античность

Индустриальная эпоха

Космическая эпоха

### 14.5. Поведение эпох

**Каждая эпоха:**
- имеет собственный production loop
- имеет собственные цепочки ресурсов
- работает независимо от других
- производит рабочие циклы через CycleGenerator-здание, которые пополняют глобальный счётчик `globalWorkCycles`

### 14.5а. Рабочий цикл эпохи

**Рабочий цикл** — итоговый продукт производственной цепочки эпохи. Это не обычный ресурс в складе, а вклад эпохи в глобальный прогресс.

```text
epoch_cycle_flow:
  advanced_resources (эпоха) 
    → CycleGenerator (здание)
      → epoch_cycle_output (tier = EpochOutput)
        → globalWorkCycles (GameState, глобальный счётчик)
```

**Пример для Каменного века:**
- инструменты → Кузня циклов → "Рабочий цикл каменного века" → globalWorkCycles += N

**Пример для Индустриальной эпохи:**
- машинные детали → Промышленный агрегатор → "Промышленный рабочий цикл" → globalWorkCycles += N

### 14.6. Изоляция экономики

StoneAge.resources != Industrial.resources

**Но:**

могут взаимодействовать через связи эпох

### 14.7. Разблокировка эпох

unlock_epoch if:technology_requirements metAND global progression threshold reached

### 14.8. Добавление новой эпохи

**При открытии:**
- создаётся новый слой
- добавляются здания
- добавляются ресурсы
- включается независимая симуляция

### 14.9. Масштаб сложности

**Каждая новая эпоха:**
- увеличивает количество цепочек
- увеличивает требования
- увеличивает мультипликативные связи

### 14.10. Влияние на старые эпохи

new_epoch can modify old_epoch via bonuses

### 14.11. Прогресс внутри эпохи

**Каждая эпоха имеет:**
- собственное дерево технологий
- собственные здания
- собственные bottlenecks

### 14.12. Параллельная симуляция

**Все эпохи:**

run in parallel simulation ticks

### 14.13. Общий пул vs изоляция

**Внутри эпохи:**
- общий склад
- автоматическая логистика

**Между эпохами:**
- отдельные системы
- ограниченный обмен

### 14.14. Взаимодействие эпох

**Эпохи могут:**
- поставлять ресурсы друг другу
- усиливать производство
- открывать новые рецепты

### 14.15. Пример взаимодействия

Industrial steel -> Space rocket components

### 14.16. Ограничение взаимодействия

**Чтобы избежать хаоса:**
- кросс-эпохные связи ограничены
- есть коэффициенты ослабления
- cross_epoch_effect *= 0.3 - 0.7

### 14.17. UI требования

**Игрок должен видеть:**
- отдельные панели эпох
- текущий прогресс каждой
- ресурсы и производство
- связи между эпохами

### 14.18. Баланс

**Эпохи должны:**
- добавлять сложность
- не заменять старые системы
- создавать новые bottlenecks

### 14.19. Ошибки проектирования

Эпохи слишком независимы (нет смысла связывать)

Эпохи слишком связаны (теряется смысл слоёв)

Слишком быстрый рост сложности

Дублирование контента

### 14.20. Минимальный набор (definition of done)

**Система реализована, если:**
- есть минимум 2 эпохи
- они работают параллельно
- у каждой есть свои здания и ресурсы
- есть независимые цепочки производства
- есть базовое взаимодействие между эпохами
- у каждой эпохи есть `epochCycleResourceId` и здание CycleGenerator
- производство рабочих циклов отображается в UI и попадает в globalWorkCycles
