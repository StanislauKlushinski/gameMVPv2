# Игровой цикл (core loop)

Игровой цикл (core loop)

### 1.1. Определение

Игровой цикл — непрерывный процесс преобразования ресурсов с целью увеличения скорости производства и открытия нового контента.

Ресурсы -> Производство -> Улучшения -> Ускорение -> Новые возможности -> Повтор

### 1.2. Состояния цикла

**Цикл разбивается на повторяющиеся состояния:**
- Генерация ресурсов
- Потребление ресурсов
- Преобразование (крафт)
- Накопление
- Инвестирование (строительство/апгрейды)
- Ускорение
- Разблокировка нового контента

### 1.3. Детализация этапов

#### 1.3.1. Генерация ресурсов

**Источник:**
- добывающие здания
- производственные здания

**Формула:**

```text
production_rate = base_rate * building_multiplier * worker_multiplier * global_multiplier
```

**Где:**
- base_rate — базовая скорость здания
- building_multiplier — уровень здания
- worker_multiplier — влияние рабочих
- global_multiplier — технологии, престиж

#### 1.3.2. Потребление ресурсов

**Каждое производственное здание:**
- Условие работы:
- если (все входные ресурсы >= требуемых) -> работаетиначе -> простаивает

**Потребление:**

input_resources -= recipe_cost * delta_time

#### 1.3.3. Преобразование (крафт)

**Производство по рецепту:**

output += recipe_output * efficiency * delta_time

**Где:**
- efficiency зависит от:
- рабочих
- апгрейдов
- глобальных бонусов

#### 1.3.4. Накопление

**Все ресурсы складываются в хранилище эпохи:**
- Ограничение:
- resource <= storage_limit

**Если превышен лимит:**

```text
production = 0 (или уменьшается)
```

#### 1.3.5. Инвестирование

**Игрок тратит ресурсы на:**
- строительство зданий
- улучшение зданий
- технологии

**Формат стоимости:**

```text
cost = base_cost * growth_factor^level
```

#### 1.3.6. Ускорение

**Источники ускорения:**
- рабочие
- технологии
- глобальные бонусы
- престиж

**Общая формула:**

```text
total_multiplier = product(всех множителей)
```

#### 1.3.7. Разблокировки

**Условия:**
- накопление ресурсов
- выполнение порога
- покупка технологии

**Результат:**
- новые здания
- новые ресурсы
- новая эпоха

### 1.4. Временная модель

#### 1.4.1. Тик системы

**Игра работает по тиковой модели:**

```text
tick_rate = 0.1 - 1 сек
```

**На каждом тике:**
- обновляются все здания
- пересчитываются ресурсы

#### 1.4.2. Delta time

**Все расчёты через:**

delta_time

**Это нужно для:**
- оффлайн прогресса
- стабильности симуляции

### 1.5. Приоритеты обработки

**Порядок обновления на тике:**
- Проверка доступности ресурсов
- Потребление ресурсов
- Производство ресурсов
- Применение ограничений (лимиты)
- Обновление UI

### 1.6. Поведение при дефиците

**Если не хватает ресурсов:**

здание останавливается полностью

### 1.7. Поведение при избытке

**Если склад заполнен:**

стоп производства

### 1.8. Цикл игрока (поведение)

**Игрок постоянно делает:**
- анализ узких мест (bottlenecks)
- увеличение производства нужного ресурса
- перераспределение рабочих
- покупка апгрейдов
- открытие нового контента

### 1.9. Узкие места (ключевая механика)

**Игра строится на:**

```text
bottleneck = ресурс с минимальной скоростью производства
```

**Игрок:**
- находит bottleneck
- устраняет его
- создаёт новый bottleneck

### 1.10. Связь с другими системами

**Игровой цикл напрямую связан с:**
- Ресурсами -> что производится
- Зданиями -> как производится
- Рабочими -> скорость
- Логистикой -> доступность
- Технологиями -> разблокировки
- Престижем -> ускорение цикла

### 1.11. Требования к реализации

Все расчёты должны быть детерминированы

Система должна поддерживать большие числа (idle scaling)

Все множители должны быть прозрачны игроку

Любой простой должен быть объясним

**Пересчёт должен работать для:**
- онлайн
- оффлайн

### 1.12. Минимальная проверка (definition of done)

**Игровой цикл считается реализованным, если:**
- ресурсы генерируются автоматически
- здания потребляют и производят
- есть лимиты
- можно строить и улучшать
- есть хотя бы 1 bottleneck
- игрок может ускорять производство
