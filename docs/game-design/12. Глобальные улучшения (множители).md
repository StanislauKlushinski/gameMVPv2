# Глобальные улучшения (множители)

Глобальные улучшения (множители)

### 12.1. Определение

**Глобальные улучшения — система постоянных (или полу-постоянных) бонусов, которые:**
- усиливают все эпохи одновременно
- ускоряют общий прогресс
- формируют долгосрочную мета-экономику

### 12.2. Источники глобальных улучшений

**Глобальные улучшения получаются из:**
- престижа
- технологий высокого уровня
- специальных зданий (поздняя игра)

### 12.3. Типы глобальных эффектов

#### 12.3.1. Производственные множители

global_production *= multiplier

#### 12.3.2. Скорость производства

cycle_speed *= multiplier

#### 12.3.3. Логистика

transfer_rate *= multiplier

#### 12.3.4. Лимиты

storage_limit *= multiplierworker_limit *= multiplier

#### 12.3.5. Кросс-эпохные эффекты

cross_epoch_effects *= multiplier

### 12.4. Структура улучшений

```text
GlobalUpgrade {id: stringname: stringcost: resource | prestige_pointseffect_type: enumvalue: floatlevel: int}
```

### 12.5. Формула роста стоимости

```text
cost = base_cost * (scale ^ level)
```

### 12.6. Система уровней

**Каждое улучшение может:**
- иметь уровни
- усиливаться постепенно
- иметь кап

### 12.7. Категории улучшений

#### 12.7.1. Экономические

+% производство

-% затраты

#### 12.7.2. Логистические

скорость передачи

буферы

#### 12.7.3. Структурные

больше зданий

больше рабочих

#### 12.7.4. Мета

ускорение престижа

бонус к оффлайн прогрессу

### 12.8. Взаимодействие с престижем

prestige_multiplier -> unlocks global upgrades

### 12.9. Взаимодействие с технологиями

technologies unlock global upgrade tiers

### 12.10. Балансировка

**Глобальные улучшения должны:**
- ускорять игру
- но не заменять локальные решения
- не делать игру "автопобедой"

### 12.11. Ограничения

#### 12.11.1. Диминишинг

```text
effective_bonus = log(value)
```

#### 12.11.2. Капы

max_multiplier_per_category

### 12.12. UI требования

**Игрок должен видеть:**
- список улучшений
- текущие уровни
- эффект в цифрах
- стоимость следующего уровня

### 12.13. Ошибки проектирования

Слишком сильные мультипликаторы

Отсутствие капов

Размытые эффекты

Дублирование с технологиями

### 12.14. Минимальный набор (definition of done)

**Система реализована, если:**
- есть хотя бы 5 глобальных улучшений
- они дают ощутимый эффект
- есть рост стоимости
- они влияют на всю игру
